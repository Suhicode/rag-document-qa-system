# Suggested Commit Messages

Realistic commit messages that sound like iterative development, not AI-generated summaries.

## Good Examples (Use These)

```
Fix resume chunking edge case
```

```
Add reference resolution for follow-up questions
```

```
Align grounding validation with retrieval results
```

```
Handle PDF extraction spacing issues
```

```
Add chunking adviser for document type detection
```

```
Fix conversation memory truncation bug
```

```
Improve error messages for Ollama connection failures
```

```
Add multi-format document support (DOCX, CSV)
```

```
Reduce chunk size for resume documents
```

```
Add source de-duplication in response formatting
```

```
Fix grounding check false positives
```

```
Add Streamlit web interface
```

```
Optimize embedding generation for large documents
```

```
Fix memory adviser false positives on short questions
```

```
Add evaluation metrics for retrieval quality
```

## Bad Examples (Avoid These)

❌ `Implemented advanced AI memory adviser with intelligent query analysis`

❌ `Added comprehensive enterprise-grade RAG system with production-ready features`

❌ `Implemented sophisticated hallucination prevention mechanism using multi-layer approach`

❌ `Enhanced document processing pipeline with intelligent chunking optimization`

❌ `Added state-of-the-art conversation memory system with context-aware follow-up detection`

## Pattern

Good commit messages:
- Short (50 chars or less for subject)
- Focus on what changed, not how amazing it is
- Use past tense ("Fixed", "Added", "Changed")
- Be specific about the problem solved
- No buzzwords or marketing language
